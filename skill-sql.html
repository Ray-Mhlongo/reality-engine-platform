End to end project pipeline

SQL

I used SQL as the foundation of this project to design and build a structured relational database. It helped me organize data into multiple related tables including customers, subscriptions, packages, tickets and outages. This structure allowed me to understand how real world systems store and connect data across different entities.

I also used SQL to query the data through joins filtering and grouping. This helped me extract meaningful insights directly from the database such as customer distribution subscription behavior and support activity. These queries formed the backbone of all analysis done in later tools.

This stage represents the starting point of the entire project pipeline. It shows how raw structured data can be turned into a usable analytical system that supports reporting and decision making across multiple tools.
